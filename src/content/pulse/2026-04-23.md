### Новости
- [Chrome 148 теперь поддерживает Structured Clone для messaging](https://developer.chrome.com/blog/structured-clone-messaging?hl=en):
	- `Map`, `Set`, `Date`, `File` и `Blob` передаются между контекстами расширения без ручной сериализации
	- опционально: `"message_serialization": "structured_clone"` в manifest.json
	- расширения с разным форматом сериализации не смогут общаться между собой
- [Shadcn radio button — 260 строк кода вместо одной](https://frontendmasters.com/blog/the-incredible-overcomplexity-of-the-shadcn-radio-button/):
	- `<input type="radio">` — и это всё, что нужно
	- Shadcn: 45 строк + Radix за спиной (215 строк, 7 импортов)
	- accessibility не лучше, стилизация не проще

### Проекты
- [Wiretext](https://wiretext.app/) — wireframe-редактор, где всё рендерится Unicode box-drawing символами. Рисуешь блоки, текст, кнопки чистым текстом — идеально для быстрых набросков без GUI
- [tiks](https://github.com/rexa-developer/tiks) — процедурные звуки для веб-интерфейсов через Web Audio API:
	- осцилляторы, шум, кривые — никаких mp3, только математика
	- единая «звуковая тема», всё звучит консистентно
	- core ~2KB, React hook ~300B, уважает `prefers-reduced-motion`

### Статьи, мнения, туториалы

#### JS/TS
- [DistributiveOmit](https://www.youtube.com/watch?v=J4vJ-wO6Sl0) — как стандартный `Omit<T, K>` превращает union-типы в `{}`:
	- `keyof` для union видит только общие ключи всех веток
	- `Exclude` возвращает `never`, и тип схлопывается
	- решение: `T extends any ? Omit<T, K> : never` — применяет Omit к каждой ветке отдельно
