### Новости

• [Chrome 147](https://developer.chrome.com/release-notes/147) — border-shape (произвольные формы с тенями и рамками), contrast-color() (автоматический контрастный текст), element-scoped view transitions, Math.sumPrecise, XML-парсер на Rust. shape() уже в интеропе 2026.

• [border-shape: прощай, квадратный веб](https://css-tip.com/border-shape/) — border-shape принимает те же значения что clip-path, но сохраняет decorations: рамки, тени, аутлайны. Chrome-only (V147+). clip-path для reveal-эффектов, border-shape для статичных форм.

### AI/Agents

• [AI fatigue is real](https://siddhantkhare.com/writing/ai-fatigue-is-real) — Siddhant Khare, мейнтейнер OpenFGA: shipped больше кода чем когда-либо, но и выгорел как никогда. AI снижает стоимость production, но повышает стоимость review и координации — а эти издержки ложатся на человека. Non-determinism ломает инженерную привычку к отладке.

• [Craft is untouchable](https://www.chrbutler.com/craft-is-untouchable) — Christopher Butler: AI не угрожает мастерству — угрожает искушение пропустить итерацию. Инструмент не определяет качество, подход определяет. Синтезатор не убил музыкальный craft, AI не убьёт дизайн-craft.

• [Nullify: детект и блок AI-транскрипции](https://dev.to/dpelleri/i-built-an-open-source-app-to-detect-block-invisible-ai-meeting-transcription-5da) — Open-source десктоп-приложение для macOS/Windows: распознаёт скрытые AI-инструменты транскрипции в Zoom/Meet/Teams и блокирует через психоакустические искажения. Всё локально.

• [Feedback Flywheel](https://martinfowler.com/articles/reduce-friction-ai/feedback-flywheel.html) — Rahul Garg (Thoughtworks): четыре типа сигнала из AI-сессий — context, instruction, workflow, failure — замыкаются обратно в shared artifacts. Каждый виток оставляет инфраструктуру чуть лучше подготовленной.

### JS/TS

• [OpenUI: Rust WASM-парсер переписан на TypeScript](https://www.openui.com/blog/rust-wasm-parser) — WASM-версия проигрывала из-за overhead на передачу данных между JS и WASM. TypeScript-реализация один-в-один оказалась быстрее. WASM хорош для тяжёлых задач (видео), но не для парсинга с частыми boundary-crossings.

### HTML/Accessibility

• [Emojis и icons: screen reader accessibility](https://blog.pope.tech/2026/04/01/making-emojis-and-icons-screen-reader-accessible/) — Emoji — это picture characters с официальными названиями, не картинки. 🙏 = «Hands pressed together», не «thank you». Три метода: visible text, hidden text (sr-only), aria-label. В соцсетях — emoji в конец предложения.

• [Роли presentation, none, generic и aria-hidden](https://t.me/alexnozer_dev/357) — generic: системная роль для div/span, нельзя задавать вручную. presentation/none: сбрасывают семантику, элемент остаётся в дереве доступности. aria-hidden: полностью исключает из дерева. Не путайте.

### CSS

• [Modern CSS](https://modern-css.com/) — Визуальная коллекция примеров: что уже не нужно писать руками в 2026 году. Container queries, nesting, :has(), scroll-driven animations, anchor positioning. Полезно если ещё не пробовали.

### Performance

• [Preloading Images](https://telp.cc/1Ki4) — Стратегии предварительной загрузки критических изображений через HTML и JavaScript для улучшения perceived performance.

### Инструменты

• [The SVG: 4700+ brand icons](https://github.com/glincker/thesvg) — Open-source SVG-библиотека брендов: поиск, версии, CDN/CLI/API, типизированные пакеты для React/Vue/Svelte. Варианты: color/mono/light/dark/wordmark.

• [Под капотом нового фронтенда MDN](https://developer.mozilla.org/en-US/blog/mdn-front-end-deep-dive/) — Много веб-компонентов, Rspack вместо Webpack, Markdown для контента, современный CSS с нативной вложенностью, интерактивные демки. Всё на GitHub.
